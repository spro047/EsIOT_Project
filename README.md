# EsIOT_Project: Plant Disease Detection with Deep Learning

This repository contains materials for a 6th-semester academic project for the Embedded Systems and Internet of Things (EsIOT) course. The project focuses on "Plant Disease Detection using Deep Learning and Feature Extraction," explicitly utilizing Convolutional Neural Networks (CNNs).

## Description

The project explores agricultural applications of Deep Learning by identifying and classifying plant leaf diseases. Early and accurate detection of plant diseases is crucial for ensuring food security and maximizing agricultural yield. The repository primarily serves as a workspace for group members, containing research papers, literature survey reports, and presentation materials related to the topic. 

## Project Structure

- **Papers/**: Contains related research papers, a literature survey report (`IOT_LS_REPORT.odt`), and the main project presentation (`AI_Plant_Disease_Detection.pptx`).
- **Team Folders (`ANUSHRI`, `SHASHANK`, `SRUSHTI`, `TANVI`)**: Individual workspaces for team members to manage their specific contributions or datasets (such as the Mendeley dataset reference for leaf images).

## How to Use and Run

Since this repository currently serves as a documentation and research repository rather than a deployable software application, there is no direct code to "run."

However, to utilize the materials:
1. Clone the repository: `git clone https://github.com/spro047/EsIOT_Project.git`
2. Navigate to the `Papers/` directory to read the literature survey (`IOT_LS_REPORT.odt`) and original reference papers.
3. Open `AI_Plant_Disease_Detection.pptx` to view the comprehensive presentation on the system architecture and proposed DCNN approach.
4. Review the dataset references located in the `SRUSHTI` directory to understand the deep learning data pipeline.

## Challenges Faced

During the research and compilation phase of this EsIOT project, several challenges were encountered:

1. **Dataset Acquisition and Refinement**: Finding a diverse and massive dataset of plant leaf images (with both healthy and diseased specimens across multiple crops) was difficult. Ensuring the data was clean and well-labeled was a major hurdle.
2. **Algorithm Selection**: Researching and determining the optimal Deep Learning architecture (e.g., standard CNNs vs. 9-layer Deep Convolutional Neural Networks) that balances computational efficiency (suitable for IoT environments) with high accuracy.
3. **Collaboration and Version Control**: Coordinating research literature, massive presentation files, and individual contributions across a 4-person team using GitHub required organizational structures (hence the individual folders).
4. **Hardware Constraints Context**: Designing a conceptually sound Deep Learning model that is ultimately intended to be integrated with IoT edge devices (EsIOT context), meaning memory and processing power limitations had to be theoretically accounted for.
